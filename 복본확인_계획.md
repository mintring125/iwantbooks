# 황지중앙초등학교 도서관 복본 확인 계획

## 목적

학생들이 구입 희망 도서로 선택한 책이 **우리 학교 도서관에 이미 있는 책(복본)이 아닌지** 확인하여, 중복 구매를 방지한다.

---

## 방법 1: 독서로 DLS에서 수동 확인 (가장 간단)

### 사이트 정보
- **독서로 DLS**: https://reading.keris.or.kr
- 로그인 없이도 "우리학교 자료검색" 기능으로 장서 검색 가능

### 확인 절차
1. 독서로 사이트 접속 → **"우리학교 자료검색"** 메뉴 클릭
2. 학교 검색: **"황지중앙초등학교"** (강원특별자치도 태백시) 선택
3. 학생들이 제출한 엑셀 파일의 도서명을 하나씩 검색
4. 검색 결과가 나오면 = **이미 소장 중 (복본)**  → 구입 목록에서 제외
5. 검색 결과가 없으면 = **미소장** → 구입 가능

### 장점
- 별도 설치 없이 바로 사용 가능
- DLS 시스템의 정확한 최신 소장 정보 활용

### 단점
- 도서 수가 많으면 하나씩 검색해야 해서 시간이 오래 걸림

---

## 방법 2: DLS 장서 목록 내보내기 + 엑셀 대조 (추천)

### 절차
1. **DLS 관리자 로그인** (사서교사/담당교사 계정)
2. **장서관리 → 장서목록 내보내기** (엑셀 다운로드)
   - DLS에서 전체 소장 도서 목록 다운로드 (엑셀/CSV)
3. **학생 희망도서 엑셀**과 **소장 목록 엑셀**을 비교
4. 도서명 또는 ISBN이 일치하는 항목 = 복본

### 엑셀에서 대조하는 방법

#### VLOOKUP 활용
학생 희망도서 엑셀의 새 열에 다음 수식 입력:
```
=IF(ISNUMBER(MATCH(B3, 소장목록!A:A, 0)), "⚠️ 복본", "✅ 구입가능")
```
- `B3`: 희망도서의 도서명 셀
- `소장목록!A:A`: DLS에서 내려받은 소장 도서 목록의 도서명 열

#### 조건부 서식 (시각적 강조)
1. 희망도서 도서명 열 전체 선택
2. 홈 → 조건부 서식 → 새 규칙
3. 수식: `=ISNUMBER(MATCH(B3, 소장목록!A:A, 0))`
4. 서식: 빨간색 배경 → 복본인 도서가 빨간색으로 표시됨

### 장점
- 한 번에 전체 목록을 대조할 수 있어 빠름
- 복본 도서가 시각적으로 바로 확인됨

### 단점
- DLS 관리자 로그인이 필요
- 장서 목록 엑셀 내보내기 절차를 알아야 함

## 방법 3: 희망도서 선택 시스템에 자동 복본 확인 기능 추가 (개발 필요)

### 개요
현재 만들고 있는 웹 시스템에 **자동 복본 확인 기능**을 추가하는 방법.

### 구현 방식

#### A. 소장 도서 목록 CSV/엑셀 업로드 방식
1. 선생님이 DLS에서 장서 목록 엑셀을 다운로드
2. 웹 시스템의 **관리자 페이지**에서 해당 엑셀 업로드
3. 시스템이 도서명/ISBN을 메모리에 저장
4. 학생이 도서를 선택할 때 자동으로 소장 여부 표시

```
[ 검색 결과 카드 ]
┌─────────────────────────┐
│  📕 어린왕자             │
│  생텍쥐페리 / 문학동네    │
│  ⚠️ 이미 도서관에 있어요! │  ← 복본 경고
│  [선택 불가]             │
└─────────────────────────┘
```

#### B. 수동 도서 목록 등록 방식
1. 선생님이 관리자 페이지에서 소장 도서 목록을 직접 입력
2. 도서명 기준으로 매칭

### 구현에 필요한 추가 작업

| 항목 | 설명 |
|---|---|
| `app.py` 수정 | 소장 도서 업로드 API 추가 (`POST /api/admin/upload-library`) |
| `app.py` 수정 | 검색 결과에 복본 여부 표시 로직 추가 |
| `index.html` 수정 | 관리자 페이지에 장서 목록 업로드 UI 추가 |
| `index.html` 수정 | 검색 결과 카드에 복본 경고 배지 표시 |
| 새 파일 | `library_catalog.json` — 업로드된 소장 목록 저장 |

- 학생이 선택하는 시점에 바로 복본 여부 확인 가능

### 단점
- 추가 개발 필요
